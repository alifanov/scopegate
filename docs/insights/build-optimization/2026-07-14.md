# Build Optimization — 2026-07-14

**Scope:** `Dockerfile`, `docker-entrypoint.sh`, `docker-compose.yml`, `.dockerignore`, `next.config.ts`, `package.json` scripts, `pnpm-lock.yaml`, `pnpm-workspace.yaml`, `docker/prisma.config.ts`, `.github/workflows/darkflow-ci-gate.yml`, git churn.

## Pipeline overview

`git push master` → **(1) CI-gate** (`darkflow-ci-gate.yml`, self-hosted runner): `pnpm install --frozen-lockfile` → **build** (`prisma generate && tsc --noEmit`, DB-less) → `lint` → `test`; красный → issue `source:ci` для воркера `fix-ci-issue` (≤3 ретрая). Параллельно → **(2) Coolify**: clone → multi-stage `docker build` (`node:22-slim`): **deps** (манифесты+prisma → `pnpm install` с BuildKit pnpm-store mount) → **builder** (`COPY . .` → `prisma generate && next build` с BuildKit `.next/cache` mount, standalone output; Next 16 → Turbopack по умолчанию) → **runner** (`.next/standalone` + static + public + prisma + отдельный `npm install prisma` в `/prisma-runtime`) → старт контейнера: `prisma migrate deploy` → `node server.js`. Пайплайн зрелый: все находки прошлых 5 раундов реализованы. Остаточные неэффективности — маргинальные заметки, не issue.

## Notes (minor / not filed)

- **Дубликат Prisma CLI в runner (~25–40 MB)**: runner делает отдельный `npm install --prefix /prisma-runtime prisma@X` для `migrate deploy` на старте. Слой закэширован (зависит от `package.json`), скачивание только при смене версии. БД недоступна на этапе build, standalone-бандл не содержит CLI — архитектурно необходимо. 5-й раунд заметкой.
- **CI build-шаг — `tsc --noEmit`, не `next build`**: ловит type-ошибки быстро и DB-less, но не next-специфичные (RSC-границы, ошибки роутов) — те дойдут только до Coolify. Осознанный компромисс. Заметка.
- **CI: install → build → lint → test последовательны** в одном job. `lint`+`test` можно распараллелить (~30–60 с), но требует реструктуризации bash-хелпера `run_check`. Маргинально.
- **base image `node:22-slim`** (не alpine) — осознанный выбор: явная установка `openssl` для Prisma. Заметка.
- **`docker-compose.yml` передаёт build args** (`BETTER_AUTH_SECRET`, `ADMIN_*` и т.д.), но в `Dockerfile` нет ни одной `ARG` — эти аргументы молча игнорируются. Влияет только на локальный `docker compose build`, не на Coolify, на скорость не влияет. Мёртвая конфигурация; можно почистить при случае, но не issue.
- **Next 16 `next build` использует Turbopack по умолчанию** — отдельный флаг `--turbopack` не нужен, сборка уже на быстром бандлере. Проверено.
- **pnpm store cache в CI**: явного `actions/cache` нет, но self-hosted runner сохраняет глобальный pnpm store между запусками (install = link-only) — отдельный кэш-шаг не нужен.
