## Recurring Issues

- **SSRF в `safe-fetch.ts`** — второй аудит подряд затрагивает этот файл. В прошлый раз (задача #88, закрыта) чинили DNS-rebinding TOCTOU; сейчас — независимый обход через IPv4-mapped IPv6. Файл остаётся хрупкой точкой: регэксп-подход к классификации IP не полон. Рекомендация — перейти на `node:net`-разбор один раз и закрыть класс целиком.
- **CRON_SECRET timing (`!==`)** — отмечалось и ранее как informational; остаётся низким приоритетом (реплей по сети непрактичен), задачу не заводим.

## Recommendations

1. **SSRF IPv4-mapped IPv6** (заведено, задача #187, high) → нормализовать адрес (снять `::ffff:`/`64:ff9b::`) и/или перейти на `node:net.isIP` в `isPrivateIp`. AC: `isPrivateIp("::ffff:169.254.169.254")===true` и запрос отклоняется; unit-тест в `safe-fetch.test.ts`.
2. **Инвайт TOCTOU** (low, без задачи) → консумить токен атомарно: `updateMany({where:{token,usedAt:null},data:{usedAt}})` с проверкой `count===1` в одной транзакции с созданием пользователя. AC: параллельные запросы с одним открытым инвайтом создают ровно один аккаунт.
3. **Аудит-лог с чувствительными params** (low, без задачи) → рассмотреть редактирование крупных/чувствительных полей (тела писем) перед записью в `AuditLog`, либо задокументировать политику доступа/retention.
