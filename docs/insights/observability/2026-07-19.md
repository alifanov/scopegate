## Findings

### Инцидент: 500-ошибки на `POST /api/cron/refresh-tokens` (закрыт)

19.07.2026 08:00–10:00 UTC маршрут возвращал HTTP 500 нарастающим темпом: 10/24 вызовов в бакете 08:00–09:00, 22/24 в 09:00–10:00, полностью восстановился к 10:00–11:00 (0/24). Итого 48 error-спанов — 79% всех ошибок сервиса за 24ч.

**Root cause:** коммит `85b2f38` (feat instagram, задеплоен ~08:30 UTC) начал работать с `ServiceConnection.provider = instagram` до того, как это значение попало в Prisma enum `ServiceProvider` — любой запрос к `ServiceConnection` падал с `PrismaClientValidationError`, включая `findMany` внутри `refreshExpiringConnectionTokens()`. Исправлено коммитом `502db0d` (~09:49 UTC — enum + миграция). Подтверждено разработчиком в сообщении коммита ("чинит OAuth callback"), но по факту затронул и cron.

**Диагностический пробел:** за весь период не было НИ ОДНОГО ERROR-лога для scopegate — инцидент восстановлен только через корреляцию временных окон с git-историей, не через логи/трейсы. Заведена задача #186 (добавить try/catch + структурированный лог в cron-роут).
