# Build Optimization — 2026-06-28

**Scope:** `Dockerfile`, `docker-entrypoint.sh`, `docker-compose.yml`, `.dockerignore`, `next.config.ts`, `package.json` scripts, `pnpm-lock.yaml`, `pnpm-workspace.yaml`, `docker/prisma.config.ts`, `.github/workflows/darkflow-ci-gate.yml`, git churn. Деплой: Coolify собирает Dockerfile на пуш в `master`; CI-gate (self-hosted runner) гоняет install/build/lint/test.

## Pipeline overview

`git push master` → **(1) CI-gate** (`darkflow-ci-gate.yml`, self-hosted runner): `pnpm install --frozen-lockfile` → **build** (`prisma generate && tsc --noEmit`, DB-less) → `lint` → `test`; красный → issue `source:ci` для воркера `fix-ci-issue`. Параллельно → **(2) Coolify**: clone → multi-stage `docker build` (`node:22-slim`): **deps** копирует манифесты+prisma → `pnpm install` с BuildKit pnpm store mount → **builder** копирует node_modules + `COPY . .` → `prisma generate && next build` с BuildKit `.next/cache` mount, standalone output → **runner** копирует `.next/standalone` + static + public + prisma, отдельный `npm install prisma` в `/prisma-runtime` → старт контейнера: `prisma migrate deploy` → `node server.js`. **Обе находки прошлого раунда реализованы** (#145 `.next/cache` mount, #146 build-проверка в CI-gate присутствуют в коде). Пайплайн зрелый и хорошо оптимизирован; остаточная неэффективность — на стороне **деплой-триггера** (Coolify передеплоивает контейнер на docs-only коммиты).

## Recommendations

- **Coolify Watch Paths** → исключить `docs/**`, `*.md`, `.darkflow.d/**` из триггера деплоя; зафиксировать поведение в `CLAUDE.md` → acceptance: docs-only коммит не создаёт новый деплоймент в Coolify, а `src/**`/`Dockerfile`/`prisma/**` — создаёт (#157).

## Notes (minor / not filed)

- **Дубликат Prisma CLI в runner (~25–40 MB)**: runner делает отдельный `npm install --prefix /prisma-runtime prisma@X` для `migrate deploy` на старте. Слой закэширован (зависит от `package.json`), скачивание только при смене версии. Standalone-бандл не содержит CLI, БД недоступна на этапе build — оставлено заметкой 4-й раунд подряд.
- **CI: install → build → lint → test последовательны** в одном job. `lint` и `test` можно распараллелить (~30–60 с), но требует реструктуризации bash-хелпера `run_check`. Маргинально — заметка.
- **CI build-шаг — `tsc --noEmit`, не `next build`**: ловит type-ошибки, но не next-специфичные (RSC-границы, ошибки роутов); полный `next build` дошёл бы только на Coolify. Осознанный компромисс прошлого раунда (DB-less, быстро) — заметка.
- **`.dockerignore` корректен**: `*.md`, `docs`, `.darkflow.d`, `**/__tests__`, `*.test.ts`, `src/generated` исключены — снапшоты не бьют слой `COPY . .`.
- **pnpm store cache в CI**: явного `actions/cache` нет, но self-hosted runner сохраняет глобальный pnpm store между запусками (install = link-only) — отдельный кэш-шаг не нужен.
- **base image `node:22-slim`** (не alpine) — осознанный выбор: явная установка `openssl` для Prisma. Заметка.
