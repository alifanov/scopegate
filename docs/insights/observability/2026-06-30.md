**403** — штатное поведение: `OAuthTokenError` → revoke + in-app notification. Не проблема.

## Hypotheses

- **Project+endpoints query** — при дальнейшем росте трафика (>200k спанов/день) может превысить 200 мс; тогда рассмотреть денормализацию счётчика `endpointCount` на модели `Project`.
- **LinkedIn p99 1.6s** — базовая линия нового инструмента; следить, не уходит ли регулярно за 2s.
- **threads_publish_thread p99 7.4s** — ожидаемо для async containers, но если пользователи жалуются на ожидание — можно вернуть UX-ответ раньше с polling callback.

## Recommendations

1. **Добавить retry для Gmail** — 8 ошибок 429 за 24 ч; добавить `retry: { delaysMs: [1000, 2000, 4000] }` в PROVIDER_REGISTRY для `gmail` → issue #159.
2. **Мониторинг LinkedIn/Threads latency** — новые инструменты; установить базовую линию, алерт при p99 > 3s стабильно.
3. **Project aggregation** — при росте > 150k спанов/день проверить p99; пока действий не требуется.
