## Recurring Issues

Нет повторяющихся находок из предыдущих аудитов. Все ранее зафиксированные high/critical (SSRF в загрузке изображений/видео #64, GAQL-инъекция #65, ролевая модель «member может всё» #1243) — **устранены** и подтверждены в этом аудите:

- **SSRF**: загрузка медиа теперь идёт через `safeFetch` (`src/lib/mcp/youtube.ts`), который валидирует все A/AAAA-записи, пинит IP и перепроверяет редиректы; `image-utils.ts` удалён.
- **GAQL-инъекция**: значения, подставляемые в GAQL, ограничены zod-регэкспами (`gaqlNumericId = /^\d+$/`, enum, даты) в `src/lib/mcp/tools/google-ads.ts`.
- **Ролевая модель**: все мутирующие маршруты используют `requireProjectOwner`, чтение — `requireProjectMember` (`src/lib/project-auth.ts`).

## Hypotheses

- **CSP без `script-src`/`default-src`** — при появлении любого пользовательского HTML-сценария (rich-контент, сторонние виджеты) текущая CSP не сдержит XSS. Пока приложение полностью на React (авто-экранирование) и XSS-стоков не найдено, порога для issue нет. Кандидат на ужесточение при росте фронтенда.
- **Отсутствие явного rate-limit на `/api/auth/*`** — better-auth включает дефолтный троттлинг в production, но он не сконфигурирован явно. Следить при росте трафика логина.

## Recommendations

1. **Криптостойкая генерация API-ключей (high → #112):** ключ MCP-эндпоинта защищает доступ к Gmail/Stripe/Ads и т.п. → генерировать через `crypto.randomBytes(32).toString("base64url")`, убрать `@default(cuid())`, добавить троттлинг перебора неверных ключей. **Acceptance:** новые/перегенерированные ключи ≥128 бит энтропии без cuid; повторные неверные ключи ограничиваются по IP.
2. **Закрыть саморегистрацию (medium → #113):** задать `disableSignUp: true` и `requireEmailVerification`, провести инвайт-флоу через серверный createUser. **Acceptance:** прямой `sign-up/email` без инвайта отклоняется, инвайт-регистрация работает.
3. **(low, без issue) Ужесточить CSP** добавлением `default-src 'self'` и явного `script-src`. **Acceptance:** заголовок CSP включает `default-src`/`script-src`, приложение работает без нарушений политики.
