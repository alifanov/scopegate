# Build Optimization — 2026-06-22

**Scope:** `Dockerfile`, `docker-entrypoint.sh`, `docker-compose.yml`, `.dockerignore`, `next.config.ts`, `package.json` scripts, `pnpm-lock.yaml`, `Makefile`, `docker/prisma.config.ts`, `.github/workflows/darkflow-ci-gate.yml`. Deploy: Coolify builds the Dockerfile on push to `master`; CI-gate (self-hosted runner) runs install/lint/test.

## Pipeline overview

`git push master` → **(1) CI-gate** (`darkflow-ci-gate.yml`, self-hosted runner): `pnpm install --frozen-lockfile` → `lint` → `test`; красный → issue `source:ci` для воркера `fix-ci-issue`. Параллельно → **(2) Coolify**: clone → multi-stage `docker build` (`node:22-slim`: **deps** копирует манифесты+prisma → `pnpm install` с BuildKit pnpm store mount → **builder** копирует node_modules + `COPY . .` → `prisma generate && next build`, standalone output → **runner** копирует `.next/standalone` (~71 MB) + static + public + prisma, отдельно `npm install prisma` в `/prisma-runtime`) → старт контейнера: `prisma migrate deploy` → `node server.js`. Образ ≈ standalone + prisma CLI (~25–40 MB) + база. Прошлые раунды закрыли #91 (multi-stage+standalone), #92 (pnpm cache mount), #119 (расширенный `.dockerignore`), #120 (CI-gate появился). Остаточная неэффективность — в **холодном `next build`** (нет кэша `.next/cache` между сборками) и в **неполноте CI-gate** (нет проверки сборки/типов → type-ошибки доходят до Coolify).

## Opportunities found

| Area | Current | Proposed | Est. savings | Severity | Issue |
|---|---|---|---|---|---|
| Build output / incremental | `.next/cache` эфемерен в builder-стейдже → `next build` (Next 16) холодный на каждой Coolify-сборке (несколько в день) | BuildKit `--mount=type=cache,id=next,target=/app/.next/cache` на шаг сборки | Заметное ускорение `next build` на тёплом кэше; образ не растёт | medium | #145 |
| Wasted deploy cycles / CI | CI-gate гоняет только install/lint/test — нет `next build`/`tsc`; type/build-ошибки падают только на многоминутной Coolify-сборке (CLAUDE.md ошибочно утверждает «verifies the build») | Добавить DB-less шаг `prisma generate && tsc --noEmit` (или `next build`) в CI-gate | Type-ошибка ловится в CI за ~мин вместо красного Coolify-деплоя + фикс + пересборка | medium | #146 |

## Recurring

Нет повторов нерешённых проблем: все находки прошлых раундов (2026-06-07 #91/#92, 2026-06-14 #119/#120) реализованы и присутствуют в коде/конфиге. Заметка про «дубликат Prisma CLI в runner» переносится третий раунд подряд как **note (не issue)** — решение архитектурно необходимо (standalone-бандл не содержит CLI, БД недоступна на этапе build), выгода скромная против риска смены механизма миграций.

## Recommendations

- `Dockerfile` (builder) → добавить `--mount=type=cache,id=next,target=/app/.next/cache` на `RUN ... next build` → acceptance: повторная сборка переиспользует `.next/cache`, время `next build` на тёплом кэше < холодного, runner-образ не растёт (#145).
- `.github/workflows/darkflow-ci-gate.yml` → добавить DB-less проверку сборки (`prisma generate && tsc --noEmit` или `next build`, без `migrate deploy`) → acceptance: намеренная type-ошибка валит CI до Coolify-деплоя (#146).

## Notes (minor / not filed)

- **Дубликат Prisma CLI в runner (~25–40 MB)**: runner делает отдельный `npm install --prefix /prisma-runtime prisma@X` для `migrate deploy` на старте. Слой закэширован (зависит от `package.json`), скачивание идёт только при смене версии. Альтернатива — `COPY --from=builder` уже установленного prisma — экономит сетевой install при смене версии, но усложняет Dockerfile; оставлено заметкой третий раунд подряд.
- **CI: install → lint → test последовательны** в одном job. Можно распараллелить lint и test, экономия ~30–60 с, но требует реструктуризации bash-хелпера `run_check`. Маргинально — заметка.
- **pnpm store cache в CI**: явного `actions/cache` нет, но self-hosted runner сохраняет глобальный pnpm store между запусками (install = link-only, hardlinks), поэтому отдельный кэш-шаг не нужен.
- `.dockerignore` корректен: `*.md`, `.darkflow`, `docs`, `.darkflow.d`, `**/__tests__`, `*.test.ts` исключены — снапшоты observability/security/build не бьют `COPY . .` слой.
