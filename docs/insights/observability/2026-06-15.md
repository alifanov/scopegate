## Hypotheses

- **Скачок объёма запросов +68%** — с 10–11k до 17k за сутки. Единственный выброс такого масштаба в истории наблюдений. Следующий снапшот покажет, это разовый скачок или новый базовый уровень. Если сохранится — создать issue для расследования.
- **Threads p99 впервые в топ** — может быть связано с #107 (TCP/TLS failures → retry) или просто аномально активное использование Threads инструмента сегодня. Нужно наблюдать в следующих снапшотах.
- **Unnamed `GET`-спан** — 7-й снапшот с одним и тем же unnamed спаном SSE. Effort S — стоит добавить `http.route` атрибут.
- **LinkedIn / Twitter в топ-10** — social API инструменты впервые в latency-топ. Может коррелировать с объёмным скачком (больше вызовов = больше выборка = точнее p99).

## Recommendations

1. **#108 (critical, blocked)** — 6-й снапшот подряд. Снять блокировку: добавить `SpanProcessor` для редактирования `access_token=...` из `http.url` Threads spans.
2. **#122 (medium, new)** — Threads publish p99 = 11.9 с. Проверить `src/lib/mcp/tools/threads.ts` на количество последовательных вызовов и снизить до < 8 с total.
3. **#109 (high, blocked)** — 6-й снапшот. Meta Graph API 400s. Снять блокировку.
4. **Unnamed `GET`-спан** — добавить `http.route` атрибут во всех SSE handlers; effort S.
5. **Следить за объёмом** — если 17k+ повторится завтра, создать issue для расследования источника.
