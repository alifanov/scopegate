## Recurring Issues

- **Threads publish latency** — 3+ снапшота подряд, ухудшение (11.9 с → 18.6 с). Открыт #122.

## Hypotheses

- **`/api/auth/[...all]` 500 × 4** — слишком редко для отдельного issue. Возможно связано с race condition при refresh сессии или edge-case в better-auth. Отслеживать в следующем снапшоте.
- **`SELECT Project` с агрегацией p99 = 90 мс** — ниже порога 100 мс, но при росте числа проектов индекс на `McpEndpoint.projectId` может помочь. Prisma создаёт FK-индексы автоматически — нужно проверить через `\d McpEndpoint` в psql.
- **Threads OAuth callback p99 = 1 506 мс** — задержка в token exchange с graph.threads.net (942 мс). Внешний API; если проблема будет воспроизводиться чаще — рассмотреть retry с таймаутом или кэширование.

## Recommendations

1. **#122 (Threads publish)** — ситуация ухудшается, 74% от таймаута. Рассмотреть уменьшение polling interval (сейчас вероятно 1–2 с) или ранний возврат `partial_success` при достижении 20 с вместо 25 с.
2. **#147 (LinkedIn)** — добавить sub-span трейсинг для выяснения источника задержки.
3. **#140 (MCP 500)** — стабильно ~5 ошибок/день, требует ручного расследования error_log.
4. **Threads OAuth** — если p99 token exchange пересечёт 1 с стабильно, рассмотреть кэш short-lived access token (Threads поддерживает long-lived tokens).
