## Hypotheses

- **SSE p99 с 13.1М → 1.03М мс** — возможен редеплой между Jun 13 и Jun 14, сбросивший долгие соединения. Следующий снапшот покажет, вернётся ли p99 к многочасовым значениям.
- **Объём запросов стабилизировался** — после падения 34% Jun 13, сейчас ~5.7% снижение: похоже на нормальную суточную вариацию, а не тренд. Hypothesis закрыта.
- **Google OAuth callback вышел из топ-10 второй день** — изменения в `oauth-flow.ts` (в текущем git diff) возможно улучшили latency. Проверить при следующем снапшоте с OAuth-трафиком.
- **Unnamed `GET`-span** — второй SSE-подобный span без `http.route`. Стоит проверить, добавляет ли `trace.getActiveSpan()?.setAttribute('http.route', ...)` route для всех SSE handlers.

## Recommendations

1. **#108 (critical, blocked)** — 5-й снапшот подряд. Токены Threads утекают в SigNoz в именах span. Снять блокировку и добавить `SpanProcessor` для редактирования `access_token=...` из `http.url`.
2. **#109 (high, blocked)** — 5-й снапшот. 459 ошибок 400 от Meta за 7 дней. Снять блокировку и расследовать коды 190/102 → `OAuthTokenError`.
3. **#110–#111 (medium, blocked)** — Снять блокировку при наличии ресурсов: OpenRouter cache + OAuth callback profiling.
4. **Unnamed `GET`-span** — добавить атрибут `http.route` во всех SSE handlers; низкий effort.
