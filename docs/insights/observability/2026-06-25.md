- **Google Ads latency** — `list_ad_groups` и `list_keywords` стабильно >1.2 с. Скорее всего Google Ads API отвечает медленно — TTL-кэш снизит воспринимаемую латентность без изменения логики (#156).
- **LinkedIn create_post p99 = 1 973 мс** — чуть ниже порога 2 с. Если вырастет выше — создать отдельный issue.
- **Снижение трафика −3%** — 145k vs 149k; небольшой откат после прошлой недели. Стабильно.

## Recommendations

1. **#156 (Google Ads кэш)** — добавить TTL-кэш (60–300 с) для `list_ad_groups` и `list_keywords` в `src/lib/mcp/tools/googleads.ts`. Ожидаемый результат: p99 < 500 мс при повторных вызовах.
2. **threads_publish_thread** — подождать снапшота от 2026-07-01 для итоговой оценки fix 773b2c2 (убраны retry, бюджет 8 с). p99 уже снизился на 44%.
3. **HTTP 500 on MCP** — наблюдать в следующем снапшоте. Если 0 сохранится — подтвердить устранение.
